# Spatial co-clustering of malaria and sickle cell disease across health facility catchments in Uganda

This repository contains the reproducible R workflow for the paper:

**Spatial co-clustering of malaria and sickle cell disease burden across health facility catchments in Uganda: a Bayesian shared-component disease mapping analysis**

## Repository structure

- `R/` analysis scripts, organised as an 8-step workflow
- `data/raw/` input datasets (not tracked by default)
- `data/processed/` derived analytic datasets
- `results/figures/` figures generated by the workflow
- `results/tables/` model outputs, summary tables, and supplementary tables
- `manuscript/` manuscript files and additional files
- `docs/` repository notes and documentation

## Workflow order

1. `01_q2_step1_preprocess_catchments_and_outcomes.R`
2. `02_q2_step2_spatial_weights_and_global_moran.R`
3. `03_q2_step3_bivariate_local_lisa_and_multivariate_geary.R`
4. `04_q2_step4_bayesian_shared_component_model.R`
5. `05_q2_step5_extended_shared_component_model_comparison.R`
6. `06_q2_step6_hotspot_typology_stability_and_prioritisation.R`
7. `07_q2_step7_covariate_adjusted_shared_component_model.R`
8. `08_q2_step8_uncertainty_and_residual_hotspot_interpretation.R`

## Suggested use

Run the scripts in numerical order. Update file paths inside the scripts to match your local machine or project root before execution.

## Data availability

Routine health-system data and some geospatial inputs may be subject to data-governance restrictions and may not be publicly shareable in this repository. Where data cannot be shared, provide metadata, data dictionaries, and instructions for requesting access.

## Reproducibility notes

- Record the R version used for the final analysis.
- Save package versions with `renv` or an equivalent dependency manager.
- Keep all output tables and figures generated from the scripts in `results/`.
